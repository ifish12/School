Developing for Android

Developing for Android was once a tedious and annoying thing for developers to do. Up until Android 4.0(Ice Cream Sandwich). There are a few reasons a lot of developers didn't like Android very much, and that a litte bit to do with the fact that there are so many devices which is can run on, and making your application run smoothly on all of them is a hard task(Mostly for games). Up unti Android 4.0, Android didn't really have an interface guideline. So all apps looked different, and acted different, which wasn't a very good thing for developers or users alike. But Google rectified this in Ice Cream Sandwich. Google made(in my opinion) a really nice design language(called "Holo") and it's slowly catching on and works really well. Android applications are coded in a Java-like language. You use Eclipse, the JDK and things Google has added to the Eclipse SDK to code your applications. The nice thing is that you can do this on all platoforms and publishing your application to the Google Play store only costs about $25 for a lifetime developer ID. The rules to get your application are a lot less strict that Apple's store, too.
